## 2. 개선한 부분 (버그 수정 · 신규 · UX)

### 2-1. 앱 기동 불능(흰 화면) 해결
- **firebase 패키지 미설치** — `package.json`엔 있으나 `node_modules`에 없어 번들 실패 → `npm install`로 복구
- **[SalesCard.tsx](frontend/src/components/dashboard/SalesCard.tsx) 구문 오류** — 탭 전환 삼항식이 `month ? (…) : (…) : todo ? …` 형태로 깨져 있었음(`day` 분기 조건 누락 + 기본 분기 미닫힘). 팀도 같은 버그를 고쳐 이후 머지에서 팀 버전 채택
- 부수: `styles.chartWrapper`(오타) → `chartWrap` 정정

### 2-2. 인증 — 키 없이도 동작 + 크래시 방지
- Firebase 웹 config가 없으면 `getAuth()`가 `auth/invalid-api-key`로 **로드 시점에 앱 전체 크래시** → [firebase.ts](frontend/src/lib/firebase.ts)에서 **유효 키가 있을 때만 초기화, 없으면 `auth=null`**(try/catch로 예외까지 흡수)
- [AuthContext.tsx](frontend/src/auth/AuthContext.tsx): `logout`의 `signOut(auth)`, `updateProfile`의 `auth.currentUser`를 **null 가드**로 감싸 키 없을 때도 안전
- `updateProfile`에 **`store_name`(상호) 독립 수정** 추가 — 설정 화면의 "가게 이름" 편집과 연동(기존엔 이름과 강제 동일)
- 데모 로그인: 백엔드 시드 계정 `owner@cafe.com / owner123`로 즉시 확인 가능

### 2-3. 세금 신고 초안 UX
- 기존: "초안 상세 보기"가 toast 팝업으로만 표시, "세무사 공유"는 가짜 버튼
- 개선: **전용 화면([TaxDraftDetailScreen](frontend/src/screens/document/TaxDraftDetailScreen.tsx))으로 전환** — 예상 총세액, 부가세·종소세·원천세 **세목별 금액 + 산출 근거**, 요약, 자동신고 안내. "세무사 공유" 버튼 삭제

### 2-4. 관리(Management) 화면 헤더 통일
- 밋밋한 밝은 배경 → **홈(대시보드)과 동일한 오로라 배경 + 밝은 헤더 텍스트 + 둥근 크림 시트**로 통일. 두 화면이 한 몸처럼 이어짐 → [ManagementScreen.tsx](frontend/src/screens/management/ManagementScreen.tsx)

### 2-5. 설정(Settings) 화면 신규 (P0)
- **계정·가게 정보**: 이름·가게명·업종 수정, 비밀번호 변경, 로그아웃, 회원 탈퇴 — 백엔드 `/api/v1/auth` 실연동(`PATCH /profile`, `DELETE /users/{id}`)
- **구독·결제**: Free/Pro/Business 플랜 선택 + 업그레이드 + ROI 해지방지 카드 *(결제·절감액은 데모)*
- **알림 설정**: 재고부족·단가급등 토글, 리포트 주기, 방해금지 시간대 *(저장까지 동작)*
- **화면 표시**: 글자 크기(전역 적용) + 실시간 미리보기
- **[PreferencesContext](frontend/src/preferences/PreferencesContext.tsx)**: 모든 설정을 `AsyncStorage`에 영구 저장(앱 재실행 후 유지)

### 2-6. 글자 크기 전역 적용 (웹) — 실제 동작
- [WebAppearance.tsx](frontend/src/components/WebAppearance.tsx): 앱 콘텐츠(`#app-screen`)에 CSS `zoom` + **픽셀 폭 보정(프레임폭 ÷ 배율)**으로 확대. `%` 보정이 zoom과 순환 계산돼 레이아웃이 깨지던 문제를 px 보정으로 해결
- 실제 브라우저(헤드리스 Chrome) 스크린샷으로 4단계 크기 정상 적용 검증

### 2-7. 다크모드 — 추가 후 제거
- 색 반전 필터 방식으로 구현·검증까지 했으나 **사용자 요청으로 완전 제거**. 이전에 다크로 저장된 사용자도 라이트로 복귀하도록 잔재 해제 처리

### 2-8. Git 머지 충돌 해소
- 로컬 1커밋 vs 원격 18커밋 diverge로 push 거부 → [AuthContext.tsx](frontend/src/auth/AuthContext.tsx)·[SalesCard.tsx](frontend/src/components/dashboard/SalesCard.tsx) 충돌 해소(팀 버전 채택 + 크래시 가드)
- 이후 재-머지에서 **커밋에 섞여 들어간 [firebase.ts](frontend/src/lib/firebase.ts) 충돌 마커**를 발견·정리(두 의도 병합)

## 4. 기술 메모 · 주의사항

- **인증 이중화 규칙**: `EXPO_PUBLIC_FIREBASE_API_KEY`가 없거나 `mock-`으로 시작하면 → 백엔드 로컬 로그인(`/api/v1/auth/login`, HS256 JWT). 유효 키가 있으면 → Firebase 로그인. 백엔드 `get_current_user`는 Firebase 토큰 검증 실패 시 HS256 토큰을 폴백으로 수용하므로 두 경로 모두 인증 통과.
- **데모 계정**: `owner@cafe.com` / `owner123` (백엔드 기동 시 자동 시딩)
- **실행 주소**: 프론트 웹 `http://localhost:8081`, 백엔드 API `http://localhost:8000` (`/docs` Swagger)
- **검증 방법**: `npx tsc --noEmit`, Expo 웹 번들 HTTP 200, 헤드리스 Chrome(puppeteer-core, `--no-save`)으로 로드/스크린샷 — 검증용 패키지·스크립트는 커밋 대상 아님
- **로컬 저장 키**: 설정=`simplem:preferences`, 세션=`simplem:session` (웹은 `localStorage`)
