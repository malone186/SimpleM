## 3. 화면 구현 (PRD §6)

### 하단 탭 (최종 6개)
| 탭 | 내용 |
|---|---|
| 홈(대시보드) | 웰컴 헤더 + 실시간 매출 SVG 차트 + 할 일 + 발주 모달 |
| 재고 | 재고 현황 + 안전재고 알림 + OCR 입고 초안 확인 |
| 발주 | AI 발주 초안 검토·승인 (초안-승인 2단계) |
| 챗봇 | 통합 챗봇 UX (말풍선 + 추천 프롬프트) |
| 운영 | 스케줄·급여 / 세금 (슬라이딩 세그먼트) |
| 관리 | ERP 허브 (아래 세부 화면 진입) |

### 관리 허브 세부 화면 (ERP 기능 전체)
- **재료 관리** (ERP-2) — 단가·이력 + **재료 추가 폼**
- **메뉴 관리** (ERP-3) — 레시피·원가율 + **메뉴 추가 폼**(재료 행 동적 추가)
- **판매 입력** (ERP-5) — 수량 담기 → 판매 등록
- **원가 분석** (ERP-6) — 메뉴별 원가율 진단 (숫자 화면 → 마스코트 미노출)
- **법령 검색** (ERP-11) — 검색 + RAG 결과
- **서류 자동화** (ERP-12) — 템플릿 → 초안(draft) 생성

> 관리 허브는 색색깔 카드가 세로로 겹쳐 쌓이는 **덱(deck) 레이아웃** 적용.

## 4. 마스코트 "브루(BREW)"

- 캐릭터 시트(`brew.png`)에서 **6개 포즈 크롭 + 배경 처리**
  - 투명 배경(happy/clipboard/serving/wave) + 크림 배경(hero/pouring/resting)
  - flood-fill + 외곽 침식 + 최대 연결요소 유지로 **배경 단차·잔조각 제거**
- `Brew` 컴포넌트: mood별 포즈 + idle 애니메이션(바운스/손흔들기/드립)
- **브루 등장 지도** 원칙 적용: *감정의 순간엔 브루, 판단(숫자)의 순간엔 비움. 한 화면에 하나.*
  - 홈 헤더: 상태별 표정
  - 빈 화면: 턱 괸 브루 EmptyState
  - 리포트 모달: 드립 로딩 → 클립보드 + **"— 브루 드림"** 서명
  - 발주 완료: 웃는 브루 + 하트 성취 순간

## 8. Git 병합 충돌 해결 (마지막)

- 원격에 팀원의 **백엔드 API 연동** 커밋 9개 vs 로컬 1개 → diverged
- 충돌 파일: `frontend/src/auth/AuthContext.tsx`
- **해결**: 로그인/회원가입 = 백엔드 `fetch` 버전 채택 + 내 `updateProfile`(프로필 사진/이름) 유지 → 양쪽 합침
- 충돌 마커 제거 → 타입체크 통과 → `git add` 완료 (커밋은 미실행)

> ⚠️ 이후 로그인/회원가입은 실제 백엔드(`localhost:8000`) 필요. 데모 계정(test@test.com)은 백엔드 DB 계정으로 대체됨.
