# Project Rules (프로젝트 룰)

SimpleM 프로젝트 내에서 개발을 진행할 때 공통으로 적용되는 규칙입니다.

## 1. 테스트 및 도구 사용 규칙
*   **원격 웹 브라우저 테스트 금지**: 브라우저 서브에이전트(browser_subagent)를 띄우는 등 원격 조종을 통해 웹페이지에 직접 들어가서 테스트하는 일은 절대 없도록 합니다. 웹 서비스 동작 확인은 백엔드 실행 상태 검증 및 터미널 빌드 로그 체크 등 비시각적인 영역으로 제한하며, 브라우저 화면의 최종 확인은 사용자가 직접 진행하도록 합니다.

## 2. 버전 관리 및 협업 규칙
*   **사용자 승인 없는 Git 커밋 및 푸시 금지**: 깃(Git) 관련 명령어(`git commit`, `git push` 등)는 사용자가 명시적으로 요청하거나 승인한 경우에만 수행합니다. 임의로 코드를 커밋하거나 원격 저장소에 푸시하는 행동은 절대 금지합니다.
